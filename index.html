Gerenciadores de Repositórios Remotos

O que são?

Gerenciadores de repositórios remotos são serviços que hospedam código fonte e facilitam o trabalho colaborativo em projetos de desenvolvimento.

Principais Plataformas

  • GitHub: Maior plataforma, ampla comunidade
  • GitLab: Versão community e enterprise
  • Bitbucket: Integração com stack Atlassian
  • Azure DevOps: Solução Microsoft

Vantagens

  • Backup automático do código
  • Controle de versão distribuído
  • Colaboração em equipe
  • CI/CD integrado
  • Documentação e issues

Comandos Linux Essenciais

Navegação

ls - Lista arquivos
cd - Muda diretório
pwd - Mostra diretório atual
mkdir - Cria diretório

Arquivos

cp - Copia arquivos
mv - Move/renomeia
rm - Remove arquivos
cat - Mostra conteúdo

Permissões

chmod - Modifica permissões
chown - Muda proprietário
sudo - Executa como root
ps - Mostra processos

Git - Controle de Versão

Comandos Básicos

git init - Inicia repositório
git clone - Clona repositório
git add - Adiciona arquivos
git commit - Salva alterações

Trabalhando com Branches

git branch - Lista branches
git checkout - Muda de branch
git merge - Une branches
git pull - Atualiza do remoto

Fluxo de Trabalho

  1. git pull origin main
  2. git checkout -b nova-feature
  3. Desenvolver e testar
  4. git add . && git commit
  5. git push origin nova-feature

Criando Site no GitHub Pages

1

Criar Repositório

Crie um novo repositório no GitHub com nome: seu-usuario.github.io

2

Adicionar Conteúdo

Adicione seus arquivos HTML, CSS e JavaScript ao repositório

3

Configurar Pages

Vá em Settings → Pages → Selecione a branch main como source

4

Acessar Site

Seu site estará disponível em: https://seu-usuario.github.io

Exemplo de Deploy:

# Clonar repositório
git clone https://github.com/seu-usuario/seu-usuario.github.io

# Entrar na pasta
cd seu-usuario.github.io

# Adicionar arquivos do site
echo "<h1>Meu Site</h1>" > index.html

# Fazer commit e push
git add .
git commit -m "Primeiro deploy"
git push origin main

HTTPS e DNS na Cibersegurança

HTTPS - Segurança na Web

HTTPS (HyperText Transfer Protocol Secure) é a versão segura do HTTP que utiliza criptografia SSL/TLS.

  • Autenticação: Verifica a identidade do servidor
  • Confidencialidade: Dados criptografados
  • Integridade: Previne alteração dos dados

DNS - Sistema de Nomes

DNS (Domain Name System) traduz nomes de domínio para endereços IP.

  • DNS Secured (DNSSEC): Valida respostas DNS
  • DNS over HTTPS (DoH): Criptografa consultas
  • Prevenção de spoofing: Protege contra falsificação

Importância para Segurança

  • Protege dados sensíveis
  • Previne ataques man-in-the-middle
  • Garante autenticidade do site
  • Protege contra phishing
  • Melhora SEO e confiança do usuário
